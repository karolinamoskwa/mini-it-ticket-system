Case Study – Brak dostępu do sieci

1. Opis zgłoszenia:
Użytkownik zgłasza brak dostępu do sieci WiFi na komputerze służbowym.

2. Analiza problemu
Możliwe przyczyny:
- problem z kartą sieciową
- błędna konfiguracja IP
- problem z routerem
- nieaktualne sterowniki

3. Kroki diagnostyczne:
1. Sprawdzenie połączenia z innymi sieciami
2. Restart karty sieciowej
3. Sprawdzenie adresu IP
4. Restart routera
5. Aktualizacja sterowników

4. Rozwiązanie:
Problem spowodowany nieaktualnym sterownikiem karty sieciowej.
Po aktualizacji połączenie zostało przywrócone.

5. Wnioski:
- warto sprawdzać sterowniki na wczesnym etapie diagnostyki
- problem nie wymagał eskalacji
